# Minimum Viable Product (MVP)

## Funcionalidades Essenciais (Escopo do MVP)
O MVP foi definido com foco em entregar o valor central da plataforma: permitir a criação de um ambiente virtual básico onde professores possam distribuir tarefas e alunos possam entregá-las. As funcionalidades selecionadas são:

1.  **Autenticação Básica:** Cadastro e login de usuários (professores e estudantes).
2.  **Gestão de Turmas:** Criação de turmas pelos professores e ingresso de alunos via código de convite.
3.  **Gestão de Atividades:** Criação de atividades com prazos pelos professores e submissão de arquivos (entregas) pelos estudantes.
4.  **Avaliação Simples:** Atribuição de notas numéricas às entregas pelos professores.
5.  **Mural Básico:** Postagem de avisos em texto pelo professor.

## Funcionalidades Futuras (Pós-MVP)
Estas funcionalidades foram postergadas pois, embora agreguem valor, não são estritamente necessárias para validar a premissa básica do produto no primeiro ciclo de lançamento:

1.  **Gamificação (Funcionalidade Inovadora):** Sistema de XP, níveis e rankings. Requer maior complexidade de lógica de negócios e design de interface.
2.  **Integração com Videoconferência:** Criação automática de links para aulas síncronas.
3.  **Correção de PDFs em Tela:** Ferramenta para o professor fazer anotações diretamente no arquivo do aluno sem precisar fazer download.
4.  **Fóruns de Discussão:** Espaços estruturados para debates entre alunos.
5.  **Relatórios Gerenciais Avançados:** Dashboards detalhados para coordenadores pedagógicos.

## Justificativa da Priorização
A priorização foi baseada na matriz de Valor vs. Esforço. O fluxo de "Criar Turma -> Passar Tarefa -> Entregar Tarefa -> Dar Nota" representa o núcleo do problema de gestão de sala de aula online. Sem esse fluxo funcionando perfeitamente, nenhuma outra funcionalidade fará sentido. Portanto, o MVP foca exclusivamente em garantir a estabilidade e usabilidade desse fluxo principal, minimizando o tempo de desenvolvimento (Time-to-Market) para obter feedback real dos usuários o mais rápido possível. Funcionalidades de engajamento (gamificação) e otimização (correção em tela) foram movidas para o backlog futuro.
